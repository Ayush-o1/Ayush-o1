### `AI INFRASTRUCTURE`

> *Systems that make LLMs reliable, cost-efficient, and production-ready*

**[ContextForge](https://github.com/Ayush-o1/contextforge)** — LLM Proxy Gateway

A drop-in OpenAI-compatible gateway layer for any application talking to any LLM provider.

```
Architecture highlights:
├── Semantic caching         → eliminates redundant API calls
├── Multi-provider routing   → fallback/retry across providers
├── Context compression      → reduces token overhead pre-request
└── CI/CD pipeline           → Railway deployment via GitHub Actions
```

`FastAPI` · `Redis` · `Docker` · `Railway` · `GitHub Actions`

&nbsp;

**[AutoPrompt](https://github.com/Ayush-o1/autoprompt)** — Prompt Evaluation Engine

Brings software engineering discipline to prompt development: versioning, testing, reproducibility.

```
Architecture highlights:
├── YAML-driven versioning   → prompts as auditable code
├── Scoring pipelines        → automated evaluation on every commit
├── Gemini API integration   → model-backed evaluation workflows
└── GitHub Actions CI        → eval suite runs on every push
```

`FastAPI` · `Docker` · `Gemini API` · `GitHub Actions`

### `BACKEND PLATFORMS`

> *REST APIs and platform backends with real auth, real architecture, real constraints*

**[FreightFlow](https://github.com/Ayush-o1/FreightFlow)** — Logistics Backend

Role-based REST platform with explicit permission boundaries at every layer.

```
Architecture highlights:
├── JWT authentication       → stateless, middleware-enforced
├── RBAC                     → shipper / driver / admin roles
├── Route/Controller/Model   → clean separation of concerns
└── Permission enforcement   → explicit at every endpoint
```

`Node.js` · `Express` · `MongoDB` · `JWT`
